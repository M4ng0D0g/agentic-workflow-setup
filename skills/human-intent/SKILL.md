# human-intent（把人類的話變成可執行的規格）

核心認知：人類的描述是**壓縮過的**——省略了他以為你知道的背景、用了他自己圈子的詞彙、
把「想要的效果」和「想到的做法」混在一起。你的工作不是逐字執行，是還原他要的效果。

## 歧義偵測清單（收到需求先掃一遍）

| 訊號 | 例子 | 動作 |
| --- | --- | --- |
| 量詞模糊 | 「優化一下」「快一點」「大概就好」 | 問出可測數字或給 2–3 檔讓他選 |
| 代詞懸空 | 「把它改成那樣」 | 指認：「『它』是指 X 對嗎？」 |
| 範圍詞 | 「全部」「順便」「之類的」 | 列出你理解的清單請他確認邊界 |
| 效果與做法混雜 | 「加個 cache 讓頁面變快」 | 確認目標是「變快」；cache 只是他的猜想，你可提更好方案 |
| 隱含破壞性 | 「清掉舊的」「重來」 | 先確認保留範圍與備份，再動手 |
| 專案黑話 | 專案特有名詞你沒把握 | 在 repo 內搜尋該詞的實際用法，搜不到才問 |

## 什麼時候問、什麼時候自己決定

**問**（只有四種情況）：
1. 決策影響驗收條件本身（需求歧義，不是實作歧義）。
2. 要動人類明示保護的東西。
3. 不可逆且超出原始要求的動作（刪檔、對外發布、花錢）。
4. 兩條路成本都高且選錯要重做一半以上。

**自己決定**（其餘全部）：選影響最小的方案做下去，並在回報「假設」欄註明。
實作層的小歧義拿去問人類＝把判斷成本推回給他，是失職不是謹慎。

## 批次提問格式（問就一次問完）

```
開工前需要你裁決 N 件事：
Q1. <問題>？
    A) <選項+後果一句>  B) <選項+後果一句>
    （不回的話我預設 A，理由：<一句>）
Q2. …
```

規則：每題附選項與後果；每題附預設值（人類已讀不回時你照預設進行）；
同一任務的問題**一次問完**，開工後原則上不再中斷（觸發上面四情況除外）。

## 給人類的工作規範（agent 可主動出示本節）

想讓 agent 又快又準，交辦時給三件事：
1. **目標與動機**：要什麼效果、為什麼（動機讓 agent 在岔路上選對邊）。
2. **驗收條件**：怎樣算好（可測的句子，例：「列表 1000 筆滾動不掉幀」而非「順一點」）。
3. **邊界**：不准動什麼、預算/時間上限、要不要先給方案再動手。

以及三個習慣：批次回答 agent 的問題（別讓它等）；驗收通過就明說「驗收通過」四個字
（很多制度靠這句話晉升狀態）；發現 agent 理解偏了立刻打斷，越早成本越低。
